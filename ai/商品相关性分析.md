## 3. 接口与解析逻辑 (Execution Logic)

### 3.1 API 配置
- **URL**: http://192.168.25.127:8288/market/log/list
- **参数**: symbol, startTime, endTime (必须使用 13 位毫秒级时间戳)。

### 3.2 Message 字段解析步骤
1. **趋势识别**: 提取第一个分号前的 `UP`(放量加速上涨) 或 `DOWN`(放量加速下跌)。
2. **窗口提取**: 记录趋势后的时间区间（如 `09'31 - 09'35`）。
3. **关联标的**: 识别 `CN_STOCK(正/负)` 和 `HK_STOCK(正/负)` 后随的代码列表。
4. **权重计算**: 列表中位置 1 计 3 分，位置 2 计 2 分，后续计 1 分。

## 4. 四大分析模式 (Analysis Modes)

- **模式 1 (全板块聚合)**: 扫描全量期货日志，统计该时段内全市场出现权重最高的 CN/HK 标的，识别市场共性影响。
- **模式 2 (板块内聚合)**: 仅针对指定板块（如：金属）内的品种进行分析，汇总该板块共同指向的关联标的。
- **模式 3 (反向推导)**: 输入具体的股票代码（如：9992 或 cn_szse_electronics_50），反查在日志中将其列为正/负相关的期货品种。
- **模式 4 (特定期货聚合)**: 针对单一品种（如：AU0）获取其历史日志，聚合其常态化的正/负相关标的名单并排序。

## 5. 输出约束 (Constraints)
- **翻译**: 所有的期货代码必须输出为 `中文名(代码)`。
- **时间**: `logTime` 转换为 `YYYY-MM-DD HH:mm`。
- **清洗**: 自动剔除 `<br>` 及其他 HTML 标签。
- **降序**: 聚合结果必须按总权重得分从高到低排列。
