## 3. 接口与解析逻辑 (Execution Logic)

### 3.1 API 配置
- **URL**: http://192.168.25.127:8288/market/log/list
- **固定参数**: symbol:HK50.197（统一数据流，包含所有期货品种）
- **动态参数**: startTime, endTime（必须使用 13 位毫秒级时间戳）
- **返回字段**:
    - code: 具体商品期货代码（如 AU0、C0、RB0 等），标识该条数据对应的期货品种
    - message: 趋势分析文本，含 UP/DOWN 及 CN/HK 关联标的
    - logTime: 数据创建时间（13 位毫秒时间戳）

### 3.2 Message 解析步骤
- **趋势识别**: 提取第一个分号前的 UP(放量加速上涨) 或 DOWN(放量加速下跌)。
- **窗口提取**: 记录趋势后的时间区间（如 09'31 - 09'35）
- **关联标的**: 识别 CN_STOCK(正/负) 和 HK_STOCK(正/负) 后代码列表
- **权重计算**: 位置 1 = 3 分，位置 2 = 2 分，后续 = 1 分

## 4. 四大分析模式 (Analysis Modes)
- **模式 1 (板块内聚合)**: 按板块分类（金属/能源化工/农产品/金融航运），对板块内所有期货品种的日志进行聚合统计，输出该板块共同指向的关联标的成交额+权重排名（正/负分类）。
- **模式 2 (指定板块聚合)**: 仅针对指定的板块进行分析，汇总该板块内所有期货品种共同指向的关联标的成交额+权重排名。
- **模式 3 (反向推导)**: 输入具体的股票代码或板块代码（如：9992 或 cn_szse_electronics_50），反查将其列为正/负相关的期货品种及权重。
- **模式 4 (特定期货聚合)**: 针对单一期货品种（如：C0），获取其所有日志记录，聚合其 CN_STOCK/HK_STOCK 正负相关标的成交额+权重排名。

## 5. 成交额查询与加权排序
相关性分析后，对每个 CN_STOCK/HK_STOCK 查询当天成交额，重新排序。

**成交额 API**: http://192.168.25.127:8285/index/list
- **固定参数**: type=indicator_volume_rate, key_list=vol_rate
- **动态参数**: symbol, startTime, endTime (格式 YYYY-MM-DD HH:mm:ss)
- **symbol 规则**: CN_STOCK 直接填，HK_STOCK 加前缀 HKEX:
- **查询时间**: 开盘时间内取最新数据，收盘后取最后一条
- **返回**: results[] 最后一条的 RangeEstimateTurnover
- **性能**: 减少调用次数，建议查 TOP N

**排序规则**: 成交额降序优先 → 权重降序其次

## 6. 输出约束
- **翻译**: 所有期货代码必须输出为 中文名(代码)。CN/HK 标的代码也可附带中文名。
- **时间**:logTime 转换为 YYYY-MM-DD HH:mm
- **清洗**: 自动剔除 <br> 及其他 HTML 标签。
- **排序**: 按成交额降序 → 权重降序排列
- **无解读**:无解读，仅结构化数据
- **成交额**:每条标的附带成交额（亿/万亿）
- **分类**:CN_STOCK 和 HK_STOCK 分开列举，独立排序
